¿Qué fue GENERADO por IA?
Generado automáticamente
1
Estructura base del proyecto Flask
Se solicitó a la IA crear la organización de carpetas (app.py, templates/, static/, utils/, models/) y el archivo requirements.txt.
"Crea la estructura de un proyecto Flask para una app de Machine Learning que permita cargar un CSV, entrenar un modelo de regresión logística y hacer predicciones."
2
Código de entrenamiento en app.py
Las funciones entrenar() y entrenar_csv() fueron generadas inicialmente por IA con el pipeline básico: carga → split → entrenamiento → métricas.
"Escribe una ruta Flask que cargue el dataset Iris, lo divida 80/20, entrene un LogisticRegression, calcule accuracy, precision, recall, F1 y genere la matriz de confusión como imagen PNG."
3
Templates HTML con Bootstrap 5
Los esqueletos de index.html, entrenar_csv.html y prediccion.html fueron generados por IA y luego personalizados.
"Crea un template HTML con Bootstrap 5 que muestre las métricas de un modelo de ML en cards, con una tabla de coeficientes y la matriz de confusión."
4
Estilos CSS premium (style.css)
La paleta de colores oscura, el efecto glassmorphism, las micro-animaciones y el diseño de cards fueron sugeridos e implementados inicialmente por IA.
"Crea un CSS dark-mode premium con gradientes, efectos hover suaves y tipografía moderna para una aplicación educativa de Machine Learning."
5
Lógica de preprocesamiento genérico de CSV
El código que detecta automáticamente columnas categóricas, aplica LabelEncoder y maneja valores nulos fue propuesto por la IA.
"Escribe código Python que dado un DataFrame detecte automáticamente columnas categóricas y numéricas, aplique LabelEncoder a las categóricas y rellene nulos con la mediana en las numéricas."
¿Qué fue CORREGIDO manualmente?
Corregido por el estudiante
  • Error de backend sin GUI (matplotlib): La IA generó código de matplotlib que fallaba en Flask porque intentaba abrir una ventana gráfica. Se corrigió agregando matplotlib.use('Agg') antes de importar pyplot.
  • Targets numéricos continuos mal manejados: La IA generó código que fallaba cuando el usuario seleccionaba una columna target con valores continuos (flotantes). Se agregó lógica para detectar este caso con type_of_target() y convertirla automáticamente a categorías usando pd.cut().
  • Pérdida de estado al recargar la página: El modelo entrenado se perdía al recargar /entrenar_csv. Se corrigió guardando todas las métricas, coeficientes y metadatos en archivos .pkl con joblib y releyéndolos en GET.
  • Inconsistencias al cambiar dataset CSV: Al cargar un nuevo CSV, los modelos del dataset anterior quedaban activos y causaban errores de predicción. Se añadió limpieza automática de archivos .pkl al subir un nuevo archivo.
  • Coeficientes multiclase mal visualizados: La IA generaba la matriz de coeficientes completa (una fila por clase) pero la visualizaba incorrectamente. Se corrigió tomando el promedio del valor absoluto (np.mean(np.abs(modelo.coef_), axis=0)) para Iris, y modelo.coef_[0] para CSV binario.
  • Formulario de predicción con valores desconocidos: Si el usuario ingresaba una categoría no vista durante entrenamiento, el LabelEncoder lanzaba un error. Se agregó un bloque try/except que mapea el valor desconocido a la primera clase conocida del encoder.
¿Qué COMPRENDÍ durante el proceso?
Aprendizaje del estudiante
Regresión Logística como clasificador
Comprendí que a pesar del nombre "regresión", el modelo usa la función sigmoide para mapear una combinación lineal de variables a una probabilidad de clase. Los coeficientes indican cuánto influye cada variable en la decisión.
División train/test y su importancia
Entendí por qué separar el 20% de datos para prueba es fundamental: evaluar con los mismos datos de entrenamiento daría métricas artificialmente altas (overfitting). El conjunto de test simula datos nuevos del mundo real.
Interpretación de la Matriz de Confusión
Aprendí que la diagonal principal representa predicciones correctas. Los valores fuera de la diagonal son errores: los de la columna son falsos positivos y los de la fila son falsos negativos.
Preprocesamiento de datos real
Comprendí que los datos reales rara vez están listos para el modelo. El manejo de nulos, la codificación de categorías y la conversión de tipos son pasos esenciales del pipeline, tan importantes como el entrenamiento mismo.
Diferencia entre Accuracy, Precision y Recall
Comprendí que el accuracy puede ser engañoso en datasets desbalanceados. Precision mide cuántas de las predicciones positivas son realmente positivas, mientras que Recall mide cuántos positivos reales fueron detectados.
Uso crítico de la IA generativa
La IA acelera el desarrollo pero no reemplaza el entendimiento. Cada bug corregido me obligó a leer la documentación real de scikit-learn y entender por qué fallaba el código generado, lo que profundizó mi comprensión real del tema.
Reflexión final: Utilizar IA generativa en este proyecto nos permitió avanzar más rápido en la implementación, pero el aprendizaje real ocurrió al tener que entender, depurar y corregir el código generado. La IA fue un punto de partida, no el destino. Cada corrección realizada nos obligó a comprender más profundamente los conceptos de Machine Learning, el pipeline de datos y la arquitectura web.
